Console Commands

When running versions of the engine compiled in debug mode and the console enabled, You can press F2 to pause the game and enter a command in the console

  • help Prints all available commands to the console
  • new Creates a new Asset or Object, can load properties. Any Class that can be loaded from a scene file can also be loaded here.
  • add_child Adds an object created with new, to another object as a child.
  • clear Clears the console
  • delete Deletes an object from the scene
  • load-scene Loads a new scene, deleting the current one
  • print_tree Prints the Scene Tree to the console

If you use || after a command you can chain it with another command 

new Object my_obj || add_child my_obj ""
Placing || at the end of the command will keep the console open, without tabbing you back to the game window

help

Displays a list of all registered console commands and their basic usage descriptions.

help

new

Instantiates a new Asset (like a Material) or an Object (like a Sprite2D). This command supports passing Loadable Values directly via a JSON-like syntax.

  • Syntax: new [ClassName] [Alias] [Property Name 1] [Property Value 1] [Property Name 2] [Property Value 2]...
new FileImage myimage filepath "path/to/my/image" || new Sprite2D mysprite image myimage || add_child mysprite ""

add_child

Parents an object to another existing object in the scene tree. If no parent is specified, it defaults to the Scene Root.

  • Syntax: add_child [Object Alias] [Parent Name]
add_child player "MainLevel"

clear

Wipes the console output history. Useful for cleaning up the view after heavy logging or tree printing.

clear

delete

Removes a specific object from the active scene tree and frees its memory.

  • Syntax: delete [ObjectName]
# Example: Removing an enemy node
delete Enemy_01

load-scene

Closes the current scene and loads a .nova scene file from the specified path.

  • Syntax: load-scene [FilePath]
load-scene Assets/Scenes/MainLevel.nova

print_tree

Outputs a visual representation of the current Scene Tree hierarchy to the console. This is essential for debugging node paths and finding the names of objects for use with the delete or add_child commands.

print_tree
