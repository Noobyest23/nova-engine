Sprite2D

Inherits: VisualInstance2D | Inherited by: None

Description

The Sprite2D node is used to display a 2D texture or image in the scene. It automatically handles texture uploading to the GPU and provides built-in support for flipping and offsetting the visual representation without affecting the node's actual transform.

By default, Sprite2D uses a BoxMesh2D and a specialized sprite shader that supports alpha blending and transparency.

flip_h / flip_v

  • Type: bool

If true, the texture is mirrored horizontally (flip_h) or vertically (flip_v). This is handled purely in the shader via the u_flip_h and u_flip_v uniforms.

offset

  • Type: Vector2

An additional displacement applied to the sprite's rendering. This is useful for aligning a texture's "feet" or base with the node's position without moving the node itself.

SetImageFromFile(path: string)

Loads an image asset from the specified file path and assigns it to the sprite. The sprite's internal size is automatically updated to match the dimensions of the new image.

// Loading a player texture at runtime
this.SetImageFromFile("Assets/Textures/player_idle.png")
