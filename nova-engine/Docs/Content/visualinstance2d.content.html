VisualInstance2D

Inherits: Object2D | Inherited by: Sprite2D

Description

The VisualInstance2D class is the base node for all 2D objects that render to the screen. It extends Object2D by adding rendering-specific logic such as materials, meshes, layering, and color tinting.

It handles the propagation of hierarchical tinting. An object's final color is a combination of its own self_tint, its tint, and the tint of all its parents in the scene tree.

tint

  • Type: Vector4

The color multiplier for this object and all of its children. Setting this to Vector4(1, 1, 1, 0) will fade out this node and its entire sub-tree.

self_tint

  • Type: Vector4

The color multiplier applied only to this specific object. Unlike tint, this value is not inherited by children.

layer

  • Type: int

Determines the draw order. Objects with higher layer values are drawn on top of objects with lower values. This is passed to the shader as u_layer.

SetMaterial(vertex: string, fragment: string)

Creates and assigns a new material to the instance using the provided file paths to shader sources.

this.SetMaterial("Shaders/default.vert", "Shaders/outline.frag")

SetShaderUniform(uniform_name: string, value: any) / GetShaderUniform() / SetShaderImgUniform(uniform_name: string, image_path: string)

Allows dynamic modification of shader variables at runtime. Supports int, float, bool, and Vector2/3/4 types.

SetShaderImgUniform() is specifically used for loading an image and setting it onto a sampler2D

// Passing a color to a custom shader uniform
this.SetShaderUniform("u_glow_color", Vector4(0.0, 1.0, 0.5, 1.0))

// Retrieving the current value
var current_glow = this.GetShaderUniform("u_glow_color")

Standard Shader Uniforms

When rendering, VisualInstance2D automatically binds the following uniforms to your material:

  • u_model: The global transformation matrix (Matrix4).
  • u_layer: The current Z-layer (int).
  • u_color: The calculated total tint (Vector4).
  • u_view / u_projection: Camera matrices (Matrix4).
